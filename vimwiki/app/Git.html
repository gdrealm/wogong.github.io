  • Set up git

  • 移除某些文件

git rm --cached "files" 仅仅取消追踪
git rm "files" 会在工作目录删除这些文件

  • 在github中永久移除文件及其历史记录

参看帮助信息 1 $git rm --cached FILENAME 这样就可以了…如果后面跟的是目录就加上个 -r 就行了… （这个操作不会删除这个文件）

git远程仓库的管理

git remote show -v
git remote show name -v
git remote add name url
git remote rm name
git remote rename oldneme newname

多个仓库可以一个一个推送，也可以在config文件中配置别名一起推送：

[remote "all"] url = git@github.com:luoshupeng/commonuseppa.git url = git@gitcafe.com:chinesedragon/commonuseppa.git url = chinesedragon@gitcd.com:/commonuseppa

git fetch 只会抓取数据，但是不会自动合并，这就是fetch与pull的区别

git pull name branch

git puush name branch

git diff采用vimdiff

http://stackoverflow.com/questions/3713765/viewing-all-git-diffs-with-vimdiff

git config --global diff.tool vimdiff git config --global difftool.prompt false git config --global alias.d difftool

Typing git d yields the expected behavior, type :wq in vim cycles to the next file in the changeset.
